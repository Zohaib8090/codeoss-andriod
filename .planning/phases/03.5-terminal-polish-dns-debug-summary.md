## Outcomes

Extended the IDE with terminal UX improvements and spent a focused session diagnosing and working around a deep Android-specific DNS resolution bug in Node.js.

## Key Deliverables

### Terminal Improvements
- **CTRL Modifier Button**: Added a stateful `CTRL` toggle button to `TerminalControlBar`. When active, the next character typed is converted to its corresponding POSIX control code (e.g. CTRL+C = `\u0003`, CTRL+Z = `\u001A`). Button highlights when active to give visual feedback.

### NPM DNS Resolution Investigation
The `npm install` command was consistently failing with `ENOTFOUND` for `registry.npmjs.org`. Root cause identified:

**Root Cause**: `libnode.so` (Node.js v24.14.1, arm64) uses the embedded `c-ares` DNS resolver which looks for `/etc/resolv.conf`. On Android, this file does not exist in the standard location accessible to app sandboxes. The system can resolve DNS fine (Android uses its own DNS stack), but Node.js cannot.

**Fix Applied**:
1. `PtyBridge.kt` now generates `/usr/etc/resolv.conf` at startup with device DNS servers (read via `getprop net.dns1/2`) + fallback servers (1.1.1.1, 8.8.8.8).
2. `PtyBridge.kt` generates `/usr/etc/dns-override.js` — a Node.js preload script that globally sets DNS servers to Google DNS (`8.8.8.8`) and monkey-patches `dns.lookup` to use `dns.resolve4`.
3. `init.sh` exports `NODE_OPTIONS="--require .../dns-override.js"`.
4. `init.sh` exports `OPENSSL_CONF="/dev/null"` to fix native Termux OpenSSL permission crashes when loading `libnode.so`.

**Current Status**: Universal DNS patch deployed successfully. The IDE's Node.js can now resolve any domain, allowing commands like `npm install` to connect to the NPM registry normally without insecure SSL flags.

## Technical Decisions

- **dns-override.js approach**: Universal DNS patch using `dns.setServers` and `dns.resolve4` instead of hardcoded IPs, ensuring full internet access for the Node environment.
- **OPENSSL_CONF=/dev/null**: Node.js was attempting to read Termux specific paths for OpenSSL causing a silent crash. Overriding this prevents the permission denied error.
